Chapter 4 — BIM-based Model Checking

Ensuring model quality and code compliance: classify the elements to check, run rules of increasing complexity, and report findings. The headline exam topic is black-box vs white-box checking systems.

Concept cards

The 4 pillars of model checking

ModelCard 1

Foundational — the structure behind this chapter.

Model Quality→Classification→Code Compliance→Reporting & Communication (BCF)

Model quality = information quality (interpretation: correctness, relevance, integrity) + data quality (completeness, consistency, timeliness, availability). It depends on the intended use cases. Design quality is only one aspect.

Classification (for checking)

ConceptCard 2

Foundational — you must select the right objects before a rule runs.

A filter selecting a subset of elements by discipline, element type, spatial position, or property. Filters combine with Boolean AND / OR / NOT. Info is explicit or implicitly derived.

Systems: OmniClass, Uniclass — a code per object. Fails if the model lacks semantics.

Black-box vs White-box checking

CompareExamCard 3

Exam: DE-Q66 — difference + major advantages/disadvantages

Black-boxWhite-box
only input & output visible; procedure hiddenchecking procedure visible & modifiable
customization limited to input paramsfurther customization possible
+ more reliable / lower error rate+ flexible for a specific task
− extensions only via the software vendor− high effort to build; risk of wrong edits
Exam answerBlack-box: only in/out visible → reliable but extensions need the vendor. White-box: internal steps transparent → flexible but costly to develop and error-prone.

Class 4 needs extra data temporarily inserted

Why semi-automated checking?

ConceptExamCard 5

Exam: DE-Q67 — semi- vs fully-automated

Regulations contain ambiguous, natural-language rules needing human interpretation; translating every code into machine rules is hard (very few codes are computable). So a human-in-the-loop stays necessary → semi-automated.

Exam answerFull automation is unreliable because codes are ambiguous / not fully computable → keep a human for interpretation.
