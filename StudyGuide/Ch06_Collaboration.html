Chapter 6 — Collaboration & Data Management

How many disciplines work on one model without chaos: synchronous (one model + locks) vs asynchronous (federated models + merging), the concurrency mechanics, the ISO 19650 Common Data Environment, and BCF for issues.

Synchronous vs Asynchronous collaboration

CompareExamCard 1

Exam: DE-Q40 · MP-Q18 — compare wrt 3 aspects — asked every year

AspectSynchronousAsynchronous
Modelone-model approachfederated discipline models
Consistency(almost) always consistentaccepted phase-wise inconsistency
Mergingnone (coordination via locks)merging needed (coordination points)
Ownershipsharedclear ownership & responsibility
Scopewithin one companyacross companies (main AEC mode)
Exam answerSynchronous = simultaneous edits on one model, kept consistent by locks, shared ownership (within a firm). Asynchronous = separate federated models merged at coordination points, clear ownership (across firms).

Concurrency control

ConceptExamCard 2

Exam: DE-Q54 · MP-Q19 — what it is + why necessary

Locking an object while it is being modified. Cycle: retrieve lock → modify → release lock. Necessary because two users editing the same object simultaneously → conflicts / inconsistencies.

Key questions granularity (atomic object? how long?) + dependent objects (lock all / none?)

Exam answerConcurrency control = locking objects during edit to prevent two users changing the same object at once in synchronous collaboration.

Model federation

DefinitionExamCard 5

Exam: DE-Q56 — what it is + which standard

Each discipline creates its own model (clear authorship); models are merged at regular intervals into a coordination/federated model; a BIM coordinator checks clashes; issues go to the responsible author (BCF); only the author changes their model. Defined in ISO 19650.

Exam answerIndependent discipline models coordinated/merged periodically into a federated model. Standard: ISO EN DIN 19650.

Common Data Environment — 4 statuses (ISO 19650)

ProcessExamCard 6

Exam: DE-Q58 — the 4 main statuses of a CDE

Exam answerWork-In-Progress → Shared → Published → Archived, with an approval gate between each. Implemented as zones/folders in the CDE.

Issue tracking — BCF

DefinitionExamCard 7

Exam: DE-Q59 (issue tracking + suitable format) · MP-Q6 (BCF + one technical detail)

BCF = BIM Collaboration Format — a vendor-independent buildingSMART standard that transports issues, not models (carries RFI/RFC). Used for cross-discipline coordination.

Technical detail (any one)

A ZIP containing an XML file (which elements are highlighted/hidden, camera position/direction/zoom, textual description) + a screenshot. Elements referenced by IFC GUIDs.

Exam answerIssue tracking = track/manage/resolve model issues; the suitable format is BCF. Technical detail: XML + screenshot in a ZIP, elements linked by IFC GUID, plus camera viewpoint.
