Chapter 10 — BIM Project Execution

The contractual & procedural spine (ISO 19650): the client's requirements (OIR→PIR→EIR) and the bidder's answer (BEP), how model maturity is specified (LOD = LOG + LOI, LOIN), and how information exchange is engineered (IDM → MVD).

Tags say what a card is. Green box = the direct exam answer.

Concept cards

EIR & BEP

ProcessExamCard 1

Exam: DE-Q60, DE-Q70 (EIR), DE-Q71 (BEP) — meaning + relationship

The request → response cycle (per phase)

EIR Employer's / Exchange Information Requirements — the client defines goals + content/purpose of models (technical, management, contractual).

BEP BIM Execution Plan — the bidder/contractor answers how they will meet the EIR.

Exam answerEIR = client's requirements (goals/expectations); BEP = the bidder's response defining how to meet them. EIR goes into the tender; the BEP is the reply.

OIR & PIR

ConceptExamCard 2

Exam: DE-Q61 · MP-Q7 — meaning + relationship

OIR (organisation)→PIR (one project)→EIR (exchange)

OIR Organisational Information Requirements — the asset owner's org-level needs (many projects).

PIR Project Information Requirements — for one specific project (derived from OIR).

Exam answerOIR = organisation-level requirements; PIR = one project's requirements, derived from the OIR. OIR → PIR → EIR.

ISO 19650

DefinitionExamCard 3

Exam: DE-Q69 — what is ISO EN DIN 19650?

The international standard for managing information over the asset lifecycle using BIM. Defines procurement, requirements, data management, the CDE + its statuses (Ch 6) and model federation. Based on British PAS 1192-2. Basis for BIM contracts worldwide.

Exam answerISO 19650 = international BIM information-management standard (from PAS 1192-2); defines CDE, statuses, roles.

LOD (= LOG + LOI) · 100–500 · LOIN

ModelExamCard 4

Exam: DE-Q41 (assign LOD to pictures) · MP-Q15 (LOIN vs LOD, 3 ISO components)

Building maturity (BIMforum) — increasing detail

LOD = LOG (geometry) + LOI (information). 100 symbol → 200 approximate → 300 specific → 350 + interfaces → 400 fabrication → 500 field-verified.

LOIN (ISO 7817) = LOG + LOI + documentation.

Exam answerLOD splits into LOG + LOI; the 3 ISO LOIN components are LOG, LOI, documentation. Assign pictures by geometric maturity (100 generic … 500 as-built).

Watch out: building LOD 100–500 ≠ CityGML LOD 0–4 (Ch 9).

IDM → MVD (information delivery)

ProcessExamCard 5

Exam: DE-Q72 — what is the IDM used for?

IDM (process + exchange req.)→MVD (IFC subset)→mvdXML

IDM (ISO 29481) describes processes and information flows — when must which data be exchanged. Contains a Process Map (BPMN) + Exchange Requirements. It is project-independent and forms the basis for the MVD (the technical IFC-subset implementation).

Exam answerIDM = specifies the process + information-exchange requirements (process map in BPMN + exchange requirements); it forms the basis for the MVD.
