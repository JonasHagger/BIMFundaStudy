Chapter 1 — Fundamentals & Use Cases

The "what and why" of BIM: a semantic 3D model as a single source of truth, the advantages it unlocks over 2D drawings, how maturity is measured (LOD = LOG + LOI), and the use cases that define what a model must contain.

Concept cards

What is BIM? + its characteristics

DefinitionExamCard 1

Exam: DE-Q1-3 (BIM definition + characteristics) · MP-Q1 context

Definition (NBIMS, US)

A BIM is a digital representation of the physical and functional characteristics of a facility, serving as a shared knowledge resource and a reliable basis for decisions across its whole lifecycle.

3 most important characteristics

① 3D geometry② Semantics (walls, doors, slabs…)③ + Additional info (material, relations)

Semantics is what lets you derive drawings, do quantity take-off, and run simulations — a dumb 3D shape cannot.

Descriptive power ladder

Exam answerBIM = digital representation of physical + functional characteristics of a facility, a shared, reliable source across the lifecycle. Characteristics: (1) 3D geometry, (2) semantics, (3) linked additional information.

Advantages of BIM · Drawbacks of 2D drawings

CompareExamCard 2

Exam: DE-Q4 (advantages) · DE-Q5 (2D drawbacks) · DE-Q38 (advantages of 3D)

+ Advantages of BIM− Drawbacks of conventional 2D
Fewer planning errors — transparency, inter-discipline coordination, single source of truthLine drawings are not machine-understandable
Fewer tendering errors — model-based quantity take-off, fewer change ordersInfo flow on drawings alone is error-prone
Transparency in execution — model-based progress monitoringDesign changes hard to track → inconsistencies found late
Early clash detection; better lifecycle & FM data; faster deliveryNo direct downstream use (simulation, analysis) → manual re-entry
Exam answerName 3–4 each. Core advantage = fewer errors + single source of truth + downstream reuse. Core 2D drawback = not computer-interpretable, so changes cause hidden inconsistencies.

LOD = LOG + LOI

ConceptExamCard 3

Exam: DE-Q9 (LOD) · DE-Q11 (LOG vs LOI) — asked repeatedly

The split

LOD=LOG (geometry)+LOI (information)

LOG Level of Geometry — how detailed the geometry must be per phase.

LOI Level of Information — which properties/semantics an object must carry.

Exam answerLOG = geometric maturity; LOI = required semantic properties. Together they form the Level of Development (details of the 100–500 scale live in Ch 10).

Watch out: this "LOD" (building maturity) is NOT the same as CityGML LOD 0–4 (Ch 9). Don't mix them.

Planning-effort curve (MacLeamy)

SketchExamCard 6

Exam: DE-Q22 — draw/label conventional vs BIM effort, impact, and cost-of-change over time

Draw this from memory: BIM shifts effort earlier, where influence is high and change is cheap

Exam answerThe BIM effort curve peaks early; the conventional one peaks late. Because influence on cost/function is high early and the cost of changes rises over time, front-loading effort (BIM) saves cost & time.

Digital Twin

DefinitionExamCard 8

Exam: DE-Q12 — define digital twin

A digital representation of a physical or immaterial object or process from the real world in the digital world — continuously updated with data from multiple sources. The counterpart may already exist or exist only in the future.

Exam answerDigital twin = live digital counterpart of a real (or future) object/process, kept in sync via data exchange.

Geometry Kernel

DefinitionExamCard 9

Exam: DE-Q8 — what is a geometry kernel?

A software component providing the data structures + operations to represent and process 3D geometry inside CAD/BIM tools. Examples: ACIS (Spatial), Parasolid (Siemens).

Exam answerThe 3D-geometry engine of a CAD system (ACIS, Parasolid). Sits below the BIM UI (layers → see Ch 2).

Classification systems

DefinitionExamCard 10

Exam: DE-Q13 — name two classification classes (see also Ch 7, Ch 10)

A standardized taxonomy giving each component a numeric code for unambiguous naming. Name two: OmniClass (US) and Uniclass (UK). Also DIN 276 (DE).

Exam answerOmniClass, Uniclass (+ DIN 276, Masterformat).
