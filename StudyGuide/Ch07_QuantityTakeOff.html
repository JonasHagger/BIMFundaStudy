CAD quantities vs LV/BoQ quantities

CompareExamCard 1

Exam: DE-Q14 — the difference — asked repeatedly

m-to-n relationship between components and tasks

CAD quantities (authoring)LV/BoQ quantities (Leistungsverzeichnis)
component-orientedtask-oriented
one component → several tasksone task → several components
Exam answerCAD quantities are component-oriented (one component, several tasks); LV quantities are task-oriented (one task, several components). → an m-to-n mapping is required.

GAEB

DefinitionExamCard 2

Exam: DE-Q15 — define GAEB + what it stands for

GAEB = Gemeinsamer Ausschuss für Elektronik im Bauwesen (since 1985). An XML-based standard for the digital exchange of BoQ / tendering / bidding / invoicing data. Independent of BIM.

Exam answerGAEB = Joint Committee for Electronics in Construction; XML standard for exchanging tender/BoQ data (sub-schemata X86/X89).

Bill of Quantities & standards

ConceptCard 3

Foundational — context for the CAD-vs-LV question.

BoQ (= Leistungsverzeichnis, LV) = a list of activities with quantities and position numbers; the most important tendering document. Software category = AVA. Standard task texts: StLB Bau.

Container DIN 18290 = BIM-BoQ container (IFC + GAEB-XML + link model)

Classification for QTO

ReferenceCard 4

Cross-cutting — also Ch 1 & Ch 10.

OmniClass, Uniclass, DIN 276 attach a code per object so components can be matched to BoQ tasks automatically (via a matchkey / classification). Without them, matching is manual.
